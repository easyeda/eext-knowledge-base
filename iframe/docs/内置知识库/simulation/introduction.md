# 仿真介绍

嘉立创 EDA 专业版自 V3.2 版本起提供仿真功能。用户可在专业版中进行电路仿真，以验证设计的正确性。该版本内置两款仿真引擎：Simulide 与 NGSpice，用户可根据需求选择适用的引擎执行仿真。

嘉立创EDA修改的 Simulide 与 NGSpice 引擎源码请访问开源仓库地址：[https://github.com/easyeda/easyeda-simulation-engine](https://github.com/easyeda/easyeda-simulation-engine)

## 适用范围

- 主要用于模拟电路与数字电路的仿真。

- 不适用于涉及模拟器件与微处理器、微控制器、DSP、FPGA 等的通信系统。

- 不支持 IBIS 模型仿真（除通过相关工具将器件仿真数据转换为 SPICE 模型数据）。

- 不支持代码开发工具的集成。

不支持上述芯片及系统仿真的主要原因如下：

- 此类系统通常需要大量时钟周期以仿真其内部状态，内存占用显著。

- 在晶体管、门电路与控制器协同仿真时，对 CPU 与存储资源要求较高。

- 更关键的是，SPICE 仿真并非用于代码驱动设备的复杂状态与过程的工具。

## 引擎差异

Simulide 引擎定位为入门级仿真，支持实时分析。

NGSpice 引擎定位为进阶级仿真，支持瞬态、交流等多种分析。
